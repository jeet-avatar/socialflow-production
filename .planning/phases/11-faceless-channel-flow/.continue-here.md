<context>
SocialFlow is a faceless content automation platform: niche → AI script → ElevenLabs TTS → Remotion video → review queue → post.

The gap being filled: there was no proper channel creation flow or per-channel home screen. Creators had no way to review videos before posting. The user's explicit requirement: "creator must review and approve every video before it posts" — auto-post requires explicit disclaimer acceptance.

Stack: FastAPI + PyMongo backend, React 18 + TypeScript + Framer Motion frontend, MongoDB (socialflow db), Redis, Celery + APScheduler.

Key patterns in the codebase:
- _col() helpers: mongodb_service.get_database()["collection_name"]
- CurrentUser: Depends(get_current_user_id) — returns supabase user_id string (or "dev-bypass" in dev)
- DEV_BYPASS_AUTH=true env var enables dev bypass in both backend and frontend
- APScheduler MongoDBJobStore manages channel crons; render_video_task is Celery
- Auth header for dev: Authorization: Bearer dev-bypass
- Frontend API base: API_BASE_URL from ../../config/api
- Frontend auth: getAuthHeaders() from ../../utils/getAuthToken
</context>
