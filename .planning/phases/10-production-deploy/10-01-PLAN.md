<objective>
Harden the CI pipeline so test failures actually gate the build. Remove three `|| true` soft-fails that currently allow broken tests to produce and push Docker images, and inject `SCHEDULER_ENABLED=false` so APScheduler cannot attempt a MongoDB connection during CI runs.

Purpose: A CI pipeline that never fails is no CI at all. The `|| true` guards were placeholder stubs from Phase 01 before the test suite existed. Now that tests exist (Phases 04-08), failing tests must break the build.

Output: Updated `.github/workflows/ci.yml` with hard-fail test steps and SCHEDULER_ENABLED env injection.
</objective>

  <action>
Edit `.github/workflows/ci.yml` with three targeted changes:

**1. Install step — hard-fail on missing requirements-test.txt:**

Current:
```yaml
      - name: Install dependencies
        run: |
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-test.txt || true
```

Replace with:
```yaml
      - name: Install dependencies
        run: |
          pip install -r backend/requirements.txt
          pip install -r backend/requirements-test.txt
```

Rationale: `requirements-test.txt` is committed (it exists). A missing file is an error, not something to silently skip.

**2. Run tests step — remove || true, add SCHEDULER_ENABLED=false env:**

Current:
```yaml
      - name: Run tests
        working-directory: backend
        run: pytest app/tests/ -v || true
        # tests expected to fail until Phase 03 creates test suite
```

Replace with:
```yaml
      - name: Run tests
        working-directory: backend
        env:
          SCHEDULER_ENABLED: "false"
          MONGODB_URI: ""
          JWT_SECRET_KEY: "ci-test-secret-not-used-in-prod"
        run: pytest app/tests/ -v
```

Rationale: SCHEDULER_ENABLED=false prevents APScheduler from attempting a MongoDBJobStore connection during pytest startup (scheduler_lifespan registers the cron jobs at app lifespan start). MONGODB_URI="" is intentional — tests use mongomock (per STATE.md). JWT_SECRET_KEY must be non-empty for the startup check in main.py to pass.

**3. Frontend test step — remove || true:**

Current:
```yaml
      - name: Run tests
        working-directory: frontend
        run: npm test -- --passWithNoTests || true
```

Replace with:
```yaml
      - name: Run tests
        working-directory: frontend
        run: npm test -- --passWithNoTests
```

Rationale: `--passWithNoTests` already handles the "no test files found" case gracefully. The `|| true` on top means actual test failures are silently swallowed. Remove it.

**Result:** The `build-images` job (which already `needs: [test-backend, lint-backend, test-frontend]`) will only run when all three upstream jobs succeed with exit code 0.
